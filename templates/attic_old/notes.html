RETRIEVAL METHODOLOGY

Tools

Tools in the MetricsGrimoire toolset have been used to retrieve the repositories with the source data for the analysis. In particular, the following tools have been used.

  • CVSAnalY

    This tool has been used to retrieve data from the git repository, and storing it in a MySQL database which is the starting point of the analysis.

The resulting database (sanitized to remove email addresses) is published for download.

Extension

The retrieved data spans from the beginning of the history of Project in the source code management repository up to the report date. The retrieval was performed on this latter date.

ANALYSIS METHODOLOGY

Techniques and details used in the analysis of data.

Commits

Changes to the source code of the project.

Affiliation

Affiliation of developers to companies and other institutions has been done based on various heuristics, some of the automatic, some other manuals. The base heuristic has bee using the domain part of the email address to identify an institution, and affiliation of the corresponding developer to it. We have also used some web pages maintained by the project itself and in some cases with information by manual search in the Internet. As a result of this process, we have classified all developers as affiliated with an organization (which we refer in this report as "company"), as non-affiliated, or as unknown (for which we couldn't decide to which company they were affiliated). We have also considered that developers may move from company to company, identifying for some of them the first or last date of work for their employers.

Non-affiliated

After doing some research, we have found some developers to be working on their own (be them volunteers or not). We consider them as non-affiliated.

Unknown affiliation

For some developers, we couldn't decide if they were affiliated to some company, or to which company in particular they were affiliated. We consider them as "with unknown affiliation".

Identification of unique developers

We have integrated, identifying duplicate identities of developers, all of the identities found. They had been consolidated to consider unique developers that could latter be affiliated to companies.

VISUALIZATION

The visualization of the data has been produced using the vizGrimoire toolset. In particular the JavaScript producing the charts and tables is a part of the vizGrimoire.JS system. It visualized data present in JSON files produced by vizGrimoire.R from the MySQL database where all the base information is stored.

CHARTS AND TABLES

Summary charts

Charts that offer a summary, or an overview, of certain aspects of the project.

  • Commits per company (monthly)

    For each company, it shows the number of commits per month for authors affiliated to that company. Only the top ten companies, by total number of commits, are shown.

  • Active authors per company (monthly)

    For each company, it shows the number of active authors per month. We count active authors as those that committed at least one commit per month, and assign them to companies following the affiliation heuristics. Only the top ten companies, by total number of authors, are shown.

  • Commits per company (pie chart)

    For each company, it shows the number of commits, during the whole history of the project, for authors affiliated to that company. Only top five companies, by total number of reviewed commits, are shown. Commits corresponding to authors for which we have not identified an affiliation are not considered (not even under the "others" tag).

  • Authors per company (pie chart)

    For each company, it shows the number of active authors performing reviewed commits during the whole history of the project. Only top five companies, by total number of authors, are shown. We count active authors as those that committed at least one commit, and assign them to companies following the affiliation heuristics. Authors for which we have not identified an affiliation are not considered (not even under the "others" tag).

  • Top authors (table)

    Top authors, ordered by number of reviewed commits for the specified period. Period is the whole history of the project, or the specified year.

Information for all repositories

Charts offering an aggregated view of the data, organized by repository.

  • Active repositories per month (evolution chart)

    Information about the monthly evolution of the number of repositories.

  • Commits per repository (monthly)

    For each repository, it shows the number of commits per month for authors in this repository. Only the top ten repositories, ordered by total number of commits, are shown.

  • Active authors per repository (monthly)

    For each repository, it shows the number of active authors per month. We count active authors as those that committed at least one commit per month, and assign them to repository. Only the top ten companies, by total number of authors, are shown.

Information per repository

Charts and tables that offer several metrics for a given repository.

  • Commits per month (evolution chart)

    Monthly evolution of the number of commits in the corresponding repository.

  • Authors per month (evolution chart)

    Monthly evolution of the number of active authors of commits to the corresponding repository.

Information for all companies

Charts offering an aggregated view of the data, organized by company.

  • Active companies per month (evolution chart)

    Information about the monthly evolution of the number of active companies. We consider a company as active during a certain month when at least a developer affiliated to that company performed a commit during that month.

  • Commits per company (aggregated) (bar chart)

    Aggregated (for the whole history of the project) number of commits per company. We assign a commit to a company when its author is a developer affiliated to that company. Only top ten companies are included (the remaining commits are not presented in this chart).

  • Authors per company (aggregated) (bar chart)

    Aggregated (for the whole history of the project) number of authors per company. We assign authors to a company when they are affiliated to that company. Only top ten companies are included (the remaining authors are not presented in this chart).

Information per company

Charts and tables that offer several metrics for a given company.

  • Commits per month (evolution chart)

    Monthly evolution of the number of reviewed commits performed by developers affiliated to the corresponding company.

  • Authors per month (evolution chart)

    Monthly evolution of the number of active authors of commits affiliated to the corresponding company.

  • Lines added/removed per month (evolution chart)

    Monthly evolution of the number of added and removed lines by commits performed by authors of reviewed commits affiliated to the corresponding company.

  • Top authors (table)

    List of the top authors affiliated to the corresponding company, ordered by the number of reviewed commits for the specified period. Period is the whole history of the project, or the specified year.
